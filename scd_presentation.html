발표 노트

2018년 9월 14일


  1. 안녕하세요, 소프트웨어 종합 설계 연구 제안서 발표를 맡게 된 박진식이라고 합니다.

    팀원들은 순서대로 전현준, Vincent Nguyen 입니다.

  2. 발표는 다음과 같은 순서로 진행 하겠습니다.

  3. 저희가 발견한 문제는 다음과 같습니다.

    최근 하드웨어의 발전과 더불어 많은 훌륭한 머신러닝 라이브러리들이 생겨나면서, 딥 러닝 기술이 보편화 되었는데요.

    Matrix / Tensor 연산은 그 중 핵심이라고 할 수 있습니다.

  5. 하지만 TensorFlow 를 포함한 현존하는 머신러닝 라이브러리들은, Matrix / Tensor 연산을 수행함에 있어 load balancing 을 효율적으로 수행할 수 없습니다.

  6. 좀 더 자세히 말씀 드리겠습니다.

    TensorFlow 에서 연산 load balancing 을 수행하는 코드를 첨부하였는데요.

    TensorFlow 에서는 위와 같이 분산 처리 시스템의 spec 을 넘겨주더라도 실제로 연산을 parallel 하게 처리하기 위해 추가적인 코드를 작성하여야 하고요.

    분산 처리 시스템의 하드웨어 성능 차이를 고려하지 않기 때문에, 수동으로 수행하여야 하며 이는 보통 굉장히 어려운 일입니다.

  8. 그림에서 TensorFlow 의 load balancing 에 대한 간단한 예를 보실 수 있는데요.

    Matrix 의 연산 (덧셈, 뺄셈 등) 별로 각각 다른 worker hardware 에서 수행할 수 있는 방식입니다.

    이 방식의 문제점은, 연산 한개를 더 작은 단위로 분산시키는 것이 불가능하고,

    연산의 분산 처리를 어떻게 할지 사용자가 직접 정해줘야 한다는 데 있습니다.

  9. 이 문제를 해결하기 위해 저희가 제안하는 기법은 이것입니다.

    먼저 분산 처리 시스템의 연산 능력에 맞게, Matrix / Tensor 연산 하나를 여러 개의 Sub-Matrix 연산으로 분리할 계획이고요.

    이 분리 작업은 HEFT 와 같은 heuristic 한 scheduling algorithm 에 의해 이루어집니다.

    또한 network communicating cost 와 같은 factor 도 고려하여, 결과적으로 가장 최적화된 (== 시간이 적게 드는) Matrix / Tensor 연산 방법을 제안하겠습니다.

  10. 이 프로젝트는 Julia Programming Language 를 사용하여 진행될 것입니다.

    Julia Programming Language 를 사용하는 이유는, 빠른 연산 속도를 잃지 않으면서도 Python, Ruby 등과 같은 scripting 언어가 주는 편안한 개발 경험을 제공하기 때문입니다.

  11. 실험은 기본적으로, Cloud 에 존재하는 여러 대의 다른 computing power 를 가진 server 들을 사용해 이루어집니다.

    저희 라이브러리 (가칭 CMM.jl)와, 현존하는 여러 가지의 딥 러닝 라이브러리 (Tensorflow, Pytorch) 와의 benchmark 비교를 통해 알고리즘 보완 및 개선 작업을 해 나갈 계획입니다.
